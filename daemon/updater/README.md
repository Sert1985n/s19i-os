# updater

Этот модуль отвечает за **штатное обновление прошивки через Web UI**.

## Пользовательский поток

1. Открыть `System -> Firmware`.
2. Загрузить пакет прошивки в браузере.
3. Backend сохраняет пакет в staging.
4. Проверяются модель, версия, checksum, свободное место и совместимость control board.
5. Майнер переводится в maintenance mode.
6. Применяется пакет обновления.
7. Выполняется reboot и health-check после старта.

## Что должен делать реальный apply-script под S19i

- остановить hashing и фоновые сервисы
- сохранить конфиг пулов и настроек
- проверить, что пакет подходит для `Antminer S19i` и нужного типа платы
- распаковать rootfs overlay или полный update package
- обновить Web UI, API и демоны
- записать статус обновления
- инициировать controlled reboot

## Важно

Для пользователя штатный путь должен быть именно через Web UI.
SD recovery и ручной rescue оставляются только как аварийная страховка, а не как основной способ установки новых версий.
