# Web UI Firmware Update

## Цель
Сделать для S19i такой сценарий, при котором пользователь обновляет систему только из панели:

- `System -> Firmware`
- `Upload package`
- `Validate`
- `Apply update`
- `Reboot`

## Реальные ограничения
Если майнер ещё на стоковой Bitmain-прошивке, первый переход на кастомную систему может зависеть от типа control board и того, принимает ли текущая стоковая прошивка кастомный пакет.

Поэтому Web UI update должен считаться **основным путём после установки нашей системы**, а не обещанием, что любой стоковый S19i всегда примет кастомную прошивку напрямую.

## Минимальный формат пакета
Рекомендуемый вид архива:

```text
s19i-os-0.2.1.tar.gz
├─ manifest.json
├─ rootfs-overlay/
│  ├─ usr/bin/
│  ├─ etc/init.d/
│  ├─ www/
│  └─ opt/s19i-os/
└─ hooks/
   └─ post_apply.sh
```

## manifest.json

```json
{
  "package_name": "S19i-OS",
  "version": "0.2.1",
  "target_model": "Antminer S19i"
}
```

## Проверки перед применением
- модель = `Antminer S19i`
- размер файла и свободное место
- hashing остановлен

## Применение
1. перевести GUI в `maintenance mode`
2. остановить miner
3. распаковать пакет в staging
4. применить overlay
5. обновить версии и статус
6. при необходимости перезагрузить устройство
