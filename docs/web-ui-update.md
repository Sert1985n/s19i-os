# Web UI Firmware Update

## Цель

Сделать для S19i такой сценарий, при котором пользователь обновляет систему только из панели:

- `System -> Firmware`
- `Upload package`
- `Validate`
- `Apply update`
- `Reboot`

## Реальные ограничения

Если майнер ещё на стоковой Bitmain-прошивке, первый переход на кастомную систему может зависеть от типа control board и того, принимает ли текущая стоковая прошивка кастомный пакет. Поэтому Web UI update должен считаться **основным путём после установки нашей системы**, а не обещанием, что любой стоковый S19i всегда примет кастомную прошивку напрямую.

## Минимальный формат пакета

Рекомендуемый вид архива:

```
s19i-os-0.2.0.tar.gz
├─ manifest.json
├─ overlay/
│  ├─ usr/bin/
│  ├─ etc/init.d/
│  ├─ www/
│  └─ opt/s19i-os/
└─ postinst.sh
```

## manifest.json

```json
{
  "name": "S19i-OS",
  "version": "0.2.0",
  "model": "Antminer S19i",
  "board_type": ["xil", "bb", "aml", "cv"],
  "min_current_version": "0.1.0",
  "checksum": "sha256:..."
}
```

## Проверки перед применением

- модель = `Antminer S19i`
- совместимый тип control board
- размер файла и свободное место
- контрольная сумма
- температура в безопасной зоне
- hashing остановлен

## Применение

1. перевести GUI в `maintenance mode`;
2. остановить miner;
3. распаковать пакет в staging;
4. выполнить `postinst.sh`;
5. обновить версии и статус;
6. перезагрузить устройство.
